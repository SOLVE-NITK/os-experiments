Theory

The algoritm is as follows:

We have the maximum, allocated given for each process in the set of processes.Now we need available. If the value is given then we can directly use it else we need to calculate it. To calculate the value we need to have the maximum instances of each resource that the system has. Then we need to subtract the maximum number of instances of a resource by the sum of all the instanaces of the resource in allocated of processes.

Then we start with calculation of need. We calculate need for eac of the resource as :
Need = Maximum - Available
When we have completed the calculation of the need for each resource for each and every process, we start with the finding the safe sequence for execution of the process.

To check whether a process can be executed selected from the set of unexecuted processes in the system we apply the following set of equations.
  1. Work = Available
  2. Need <= Work
  3. Work = Work + Allocation

We continue this with the processes in the set of processes as long as either of the two situations arises.
Case 1
All the processes have completed their execution.
Case 2
Need > Available ; for all processes in the set of remaining processes

If the case is Case 1, then we get the safe sequence of execution but if the case is Case 2, then the system is not in safe state.
